# Fluid Webapp - Quick Start Guide\n\n## 5-Minute Setup\n\n### 1. Prerequisites Check\n```bash\njava -version      # Should be 17+\nmvn -version      # Should be 3.8+\nnode -version     # Should be 18+\n```\n\n### 2. Clone & Setup\n```bash\ncd /Users/kusumathatavarthi/jira_ai_chatbot_artifacts/fluid_webapp\nchmod +x setup.sh\n./setup.sh\n```\n\n### 3. Configure Environment\n```bash\n# Edit .env file with your keys:\n# OPENAI_API_KEY=sk-...\n# JIRA_API_TOKEN=...\n# GITHUB_PAT=ghp_...\n```\n\n### 4. Start Application\n```bash\nchmod +x start.sh\n./start.sh\n```\n\n### 5. Access URLs\n- **Frontend**: http://localhost:3000\n- **Backend**: http://localhost:8080\n- **API Test**: `curl http://localhost:8080/api/webhook/trigger-automation?ticketKey=TEST-1&repoUrl=https://github.com/user/repo.git&branch=main&description=Fix:%20Add%20error%20handling`\n\n## Key Files\n\n| File | Purpose |\n|------|----------|\n| `pom.xml` | Maven config with Spring AI dependency |\n| `src/main/java/...CodeSynthesisService.java` | LLM code generation |\n| `src/main/java/...PromptEngineer.java` | Prompt builders |\n| `src/main/resources/application.properties` | Spring AI config |\n| `frontend/src/App.jsx` | React UI for automation |\n| `SPRING_AI_GUIDE.md` | Detailed integration guide |\n\n## Testing the Workflow\n\n### 1. Manual Trigger\n```bash\ncurl -X POST \"http://localhost:8080/api/webhook/trigger-automation\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"ticketKey\": \"PROJ-1\",\n    \"repoUrl\": \"https://github.com/user/test-repo.git\",\n    \"branch\": \"main\",\n    \"description\": \"Fix: Add null safety checks to UserValidator\"\n  }'\n```\n\n### 2. Via Frontend\nNavigate to http://localhost:3000 and fill in:\n- Ticket Key: `PROJ-123`\n- Repository URL: `https://github.com/user/repo.git`\n- Description: `Fix: Add validation to payment processing`\n\nClick \"Trigger Automation\"\n\n## Common Issues\n\n### \"OpenAI API key not configured\"\n→ Set `OPENAI_API_KEY` environment variable\n\n### \"Port 8080 already in use\"\n→ Change in `application.properties`: `server.port=9090`\n\n### \"Git clone fails\"\n→ Ensure GitHub PAT has \"repo\" scope and is not expired\n\n### \"Generated code has syntax errors\"\n→ Improve ticket description with more context\n\n## Next: Production Deployment\n\n1.
